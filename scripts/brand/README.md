# Kaimoku brand asset pipeline

Canonical home for Kaimoku brand asset generation. Outputs are committed to:

- `src/app/{icon.svg,apple-icon.png,favicon.ico}` — Next.js auto-discovery
  for the deployed site's own favicons.
- `public/brand/v1/` — versioned public URLs at
  `https://kaimoku-website.vercel.app/brand/v1/*` for cross-product consumption.

> Production domain `kaimoku.tech` is the eventual target (per Round 07 brand spec) but may not yet be DNS-mapped to the Vercel deploy. Verify with `dig +short kaimoku.tech` before switching consumer URLs.

## Source of truth

| Layer  | What                                                    |
| ------ | ------------------------------------------------------- |
| Design | Figma Make file `DcxgH5TovNJbvWC7penAxp`                |
| Code   | `spec.mjs` — constants extracted from `App.tsx` by hand |
| Build  | `generate.mjs` — renders `spec.mjs` → SVG / PNG / ICO   |

`spec.mjs` is downstream of Figma but upstream of the generator. Figma changes
during design rounds; `spec.mjs` updates only when a round is locked.

## Sync from Figma

When you've updated the Figma file and want to bring `spec.mjs` in line:

1. Open a Claude Code session.
2. Ask Claude: **"sync brand spec from Figma"**.
3. Claude reads `sync-from-figma.md` and follows the workflow there: pulls
   `App.tsx` via Figma MCP, compares against `spec.mjs`, proposes a diff.
4. Review the diff. Approve or redirect.
5. On approval, Claude updates `spec.mjs` and runs `npm run brand:generate`.
6. Review the resulting changes in `src/app/` and `public/brand/`. Commit.

## Generate

```bash
npm run brand:generate
```

Idempotent. Generated bytes are deterministic given fixed `spec.mjs`.

## Versioning

`v1/` is the current major version. Bump to `v2/` only on a breaking visual
change (new mark, new wordmark face). Keep `v1/` live for ≥90 days after
`v2/` ships so consumers have a migration window.

`latest/` always duplicates the highest version. Consumers should pin to a
specific `v1/` URL for build determinism; `latest/` is for ad-hoc references
(marketing emails, social posts) where freshness beats stability.
